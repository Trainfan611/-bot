# 🚀 SteasHub VPN Bot — Деплой на Railway

## 📁 Структура проекта
```
steashub_bot/
├── bot.py              # Основной код бота
├── database.py         # Модуль базы данных (SQLite)
├── requirements.txt    # Python зависимости
├── Dockerfile          # Конфигурация для Railway
└── README.md           # Эта инструкция
```

## 🔧 Инструкция по деплою

### Шаг 1: Подготовка на GitHub
1. Создай новый репозиторий на GitHub (например, `steashub-bot`)
2. Загрузи все файлы из папки `steashub_bot` в репозиторий

### Шаг 2: Настройка Railway
1. Перейди на [railway.app](https://railway.app/)
2. Войди через GitHub
3. Нажми **"New Project"** → **"Deploy from GitHub repo"**
4. Выбери репозиторий `steashub-bot`

### Шаг 3: Переменные окружения
В Railway добавь переменные окружения (Settings → Variables):

| Переменная | Значение | Пример |
|------------|----------|--------|
| `BOT_TOKEN` | Твой токен бота (из @BotFather) | `123456:ABC-DEF1234...` |
| `PROXY_URL` | Прокси (опционально) | `socks5://127.0.0.1:10808` |
| `ADMIN_IDS` | ID администраторов (через запятую) | `123456789,987654321` |

> ⚠️ **Важно:** Railway не поддерживает SOCKS5 прокси по умолчанию. Если используешь прокси, убедись что он доступен извне.

### Шаг 4: Запуск
Railway автоматически соберёт Docker-образ и запустит бота.

## 📝 Примечания

- **Прокси:** Если v2rayN работает локально на Windows, на Railway он не будет доступен. Удали `PROXY_URL` или настрой удалённый прокси.
- **Токен:** Храни токен в переменных окружения Railway, а не в коде.
- **Логи:** Смотри логи в Railway Dashboard для отладки.
- **База данных:** SQLite хранится локально. Для production рассмотри PostgreSQL.

## 🎯 Функционал бота

### Для пользователей:
- `/start` — Главное меню с приветствием
- `/admin` — Админ-панель (только для ADMIN_IDS)
- **Просмотр тарифов** — Personal (150₽) и Family (400₽)
- **Личный кабинет** — Информация о подписке, устройствах, дате окончания
- **Репорты серверов** — Сообщение о проблемах (только для активных подписчиков, таймаут 15 мин)

### Для администраторов:
- `/root` — **Расширенная панель администратора**:
  - 📊 Общая статистика (пользователи, конверсия)
  - 💰 Доходы в рублях (день/неделя/месяц/год/всё время)
  - 📌 Подписчики по тарифам
  - 💳 Последние платежи (топ за 30 дней)
  - 🔄 Кнопка обновления данных
- `/admin` — Базовая админ-панель:
  - **📊 Статистика** — Общие метрики проекта
  - **💰 Доходы** — Детализация по периодам
  - **👥 Подписчики** — Количество и распределение по тарифам
  - **⚠️ Репорты** — Просмотр жалоб на серверы

## 🗄 База данных

Бот использует SQLite с таблицами:
- `users` — Пользователи
- `subscriptions` — Подписки
- `payments` — Платежи
- `server_reports` — Репорты серверов
- `server_stats` — Статистика серверов (для будущей проверки пинга)

## 📞 Поддержка
Если что-то не работает — проверь логи в Railway и убедись что:
- ✅ Токен правильный
- ✅ Бот добавлен в @BotFather
- ✅ ADMIN_IDS настроен для доступа к админ-панели
- ✅ Нет ошибок в логах

## 🔐 Настройка администраторов

1. Узнай свой Telegram ID через бота @userinfobot
2. Добавь в переменную окружения `ADMIN_IDS`:
   ```
   ADMIN_IDS=123456789,987654321
   ```
3. После перезапуска бота команда `/admin` будет доступна только указанным ID
