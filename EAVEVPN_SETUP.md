# 🔐 Настройка загрузки ключей из @EaveVPNbot

## Шаг 1: Получение API ключей Telegram

1. Перейдите на https://my.telegram.org
2. Войдите, используя ваш номер телефона (в формате +7...)
3. Перейдите в раздел **"API development tools"**
4. Создайте новое приложение:
   - **App title**: SteasHub Bot (или любое название)
   - **Short name**: steashub
   - **Platform**: Desktop
5. Скопируйте:
   - **App api_id** (число)
   - **App api_hash** (строка)

## Шаг 2: Настройка переменных окружения

Добавьте следующие переменные окружения:

```bash
# Для Railway (Settings → Variables)
TELEGRAM_API_ID=12345678           # Ваш api_id
TELEGRAM_API_HASH=abcdef123456     # Ваш api_hash
TELEGRAM_PHONE=+79991234567        # Ваш номер телефона
```

## Шаг 3: Первая авторизация

При первом запуске бот отправит код подтверждения на ваш Telegram.

### Для локальной разработки:
1. Запустите бота: `python bot.py`
2. Проверьте логи — бот попросит ввести код
3. Введите код из Telegram в консоль
4. Сессия сохранится в файле `eavevpn_session.session`

### Для Railway:
К сожалению, Railway не поддерживает интерактивную авторизацию.

**Варианты решения:**

#### Вариант A: Локальная сессия
1. Создайте сессию локально (`python bot.py`)
2. Скопируйте файл `eavevpn_session.session` в репозиторий
3. Railway использует эту сессию

#### Вариант B: Использовать только GitHub ключи
Не настраивайте переменные `TELEGRAM_*` — бот будет автоматически использовать ключи с GitHub.

## Шаг 4: Проверка работы

После настройки:
1. Запустите бота
2. Нажмите **"🌐 Обычный интернет (без белых списков)"**
3. Бот попытается получить ключ из @evavpn
4. Если не получится — использует резервный ключ с GitHub

## 🔁 Приоритет источников ключей

1. **@evavpn** (Telegram канал) — приоритет
2. **GitHub** (duckray-client/free-vless-keys) — резерв

## ⚠️ Важно

- Ключи обновляются **каждые 12 часов** (в 00:00 и 12:00 UTC)
- Храните `TELEGRAM_API_ID`, `TELEGRAM_API_HASH` и `TELEGRAM_PHONE` в секрете
- Не публикуйте файл сессии в публичном доступе

## 📞 Поддержка

Если возникли проблемы:
1. Проверьте логи бота
2. Убедитесь, что номер телефона указан в правильном формате (+7...)
3. Проверьте, что канал @evavpn доступен
